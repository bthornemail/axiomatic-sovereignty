# Axiomatic Sovereignty

Axiomatic Sovereignty is a portable, runtime-agnostic foundation for shared computational knowledge.

Knowledge empowers, but knowledge itself is not power. Knowledge becomes leverage when it is withheld. This repository is structured for people who only own their decisions and want a place to preserve, verify, translate, and distribute knowledge freely so better decisions do not require starting from nothing.

It is not a platform, product, institution, governing body, or ownership claim. It is a forkable public reference for people and communities who want to learn, build, and adapt without being locked into a proprietary runtime or centralized control structure.

## Repository Geometry

The repository has two public shapes:

- **Pentagonal base:** five subfolders that hold the reusable foundation.
- **Hexagonal narrative frame:** six root documents that orient people through who, what, when, where, why, and how.

Repo management files such as `README.md`, `.gitignore`, and other metadata files are not counted as hexagonal faces.

## Pentagonal Base

```text
1-logic-proofs/         what is true
2-structural-types/     what is shaped
3-procedural-manifests/ how it moves
4-canonical-axioms/     what protects the sovereign entity
5-social-contracts/     what a community voluntarily agrees to
```

Each layer should remain understandable on its own. Lower layers describe what must be true or well-formed. Higher layers describe how people and communities voluntarily coordinate around those truths.

## Hexagonal Narrative Frame

```text
Who    -> THE_OPTIONAL_IDENTITY_STATEMENT.md
What   -> THE_CONSTITUTIONAL_MANIFESTO.md
When   -> THE_OPERATIONAL_NARRATIVE.md
Where  -> THE_FOUNDATIONAL_CHARTER.md
Why    -> THE_COMPUTATIONAL_SINGULARITY.md
How    -> THE_PROTOCOL_OF_ADMISSION.md
```

The six root documents are written in natural language first. StrictYAML front matter is used only where explicit machine-readable anchors are needed.

## What This Is

- A public bootstrap for shared knowledge.
- A zero-dependency reference where core claims should be inspectable and reproducible.
- A five-layer encyclopedia of proofs, pure types, deterministic procedures, attested axioms, and real social contracts.
- A foundation that can be cloned, forked, translated, taught, and adapted.
- A place where authorship and provenance matter, but possession does not create ownership over the foundation.

## What This Is Not

- It is not a proprietary framework.
- It is not a runtime or application server.
- It is not an organization, corporation, absorbing body, or permission gate.
- It is not a demand that forks obey one central maintainer.
- It is not a claim over another person's mind, identity, knowledge, computation, or participation.

## How To Use This Repository

1. Read the hexagonal narrative frame to understand the intent.
2. Inspect the pentagonal base to find usable artifacts.
3. Clone, fork, translate, or adapt the pieces you need.
4. Preserve provenance when you carry knowledge forward.
5. Mark meaningful deltas when you change the foundation.
6. Return consequences, corrections, translations, or improvements where useful.

## Contribution Posture

Contributions are judged by claims and consequences, not by institutional status.

The admission path is:

```text
Submit -> Logic Gate -> Type Gate -> Procedure Gate -> Canonical Gate -> Social Gate -> Receipt
```

The gates evaluate whether a contribution is coherent, well-typed, reproducible, constitutionally aligned, and useful to the common foundation. They do not make contributors subordinate to a central body.

## Forking Posture

Forking is a sovereign act. A fork may preserve the foundation unchanged, translate it, specialize it, or disagree with it.

An unchanged copy does not become the property of the copier. A modified fork creates responsibility for the delta it introduces. Return what you learn, what breaks, what helps, what harms, and what should be corrected.

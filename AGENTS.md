# AGENTS

## What This Repo Is

This is the **Computational Constitution for the Commons** — a language-agnostic, zero-dependency, forkable reference architecture for community-governed computational projects.

It is not an implementation. It is a specification for belonging.

## Repo Structure

- `README.md` — the paper (Axiom 1, entry fold)
- `THE_OPTIONAL_IDENTITY_STATEMENT.md` — identity fold (Axiom 2)
- `THE_PROTOCOL_OF_ADMISSION.md` — rigor fold (Axiom 3)
- `THE_CONSTITUTIONAL_MANIFESTO.md` — shield fold (Axiom 4)
- `THE_OPERATIONAL_NARRATIVE.md` — personal fold (Axiom 5)
- `THE_CONSTRUCTED_BRANCHES.md` — ecosystem fold (Axiom 6)
- `THE_COMPUTATIONAL_SINGULARITY.md` — visionary fold (Axiom 7)
- `layers/` — five content layers
- `.geometric/` — optional geometric signatures
- `AGENTS.md` — this file

## Key Conventions

- Origami axioms 1-7 as epistemological foundation
- Five-layer validation stack (Logic, Types, Procedures, Canonical, Social)
- Fork-as-sovereignty, not fragmentation
- Language-agnostic encoding
- Geometry-agnostic by default
- Zero external dependencies

## Authority Rules

### Non-Authority

```text
folding is not computing
naming is not owning
forking is not fragmentation
geometry is not truth
implementation is not validation
contribution is not acceptance
the paper is not the practice
```

### Inherited Authority

```text
the paper (README.md) is the foundation
the seven root documents are the constitution
the five layers are the structure
the branches are the ecosystem
the geometric signatures are personal expression
```

### Strongest Invariant

```text
the commons belongs to no one and everyone
the constitution constrains what is possible, not what is required
forking is a sovereign act, not a failure
the social contract is enforced by trust, not law
```

## What to Be Careful About

- Do not conflate geometry with axioms
- Do not treat personal signatures as universal truth
- Do not add dependencies
- Do not add language-specific content to root documents
- Do not treat implementation as validation
- Do not treat contribution as acceptance
- Do not confuse the paper with the practice

## Modifying Canonical Files

Root documents (THE_*.md) and README.md are normative.

Changes to canonical files require:

```text
clear authority justification
consistency check with all other canonical files
verification that no invariants are violated
```

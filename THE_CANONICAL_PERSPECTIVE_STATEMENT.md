# The Canonical Perspective Statement

*Given two points p1 and p2, there is a unique fold that places p1 onto a line l.*

---

## 1. Machine Anchor

The front matter above is the machine-readable anchor for this branch. It gives downstream tools a small, explicit set of stable references without making this repository depend on a specific runtime, framework, platform, or institution.

Forks may change these fields to describe their own namespace, composition mode, layer layout, and public posture. A changed anchor creates responsibility for the changed meaning; it does not create ownership over the original foundation.

## 2. Perspective Is Optional

This document gives a branch a place to state how it sees the foundation.

The foundation does not require a person, fork, community, implementation, or runtime to adopt one identity before using the artifacts. It requires only that changes remain inspectable enough for others to understand what was preserved, translated, modified, or rejected.

## 3. Canonical Does Not Mean Centralized

Canonical means this branch offers a reference perspective that others can compare against.

It does not mean:

- one repository owns the knowledge
- one runtime owns the implementation
- one language owns the structure
- one maintainer owns the interpretation
- one fork becomes final over all other forks

Many languages can read the same structural truth, but no language owns it. Many forks can carry the same foundation, but no fork owns what was faithfully received.

## 4. The Observer Layer

The five layer folders form the usable base. They compress proofs, types, sequences, axioms, and agreements into artifacts that can be inspected and reused.

The six root documents form the narrative frame. They project the foundation outward so a reader can orient themselves by who, what, when, where, why, and how.

The seventh layer is not inside the repository. It belongs to the observer: the fork, translation, application, community, runtime, or person who carries the knowledge forward.

## 5. Minimum And Maximum Bounds

The repository sits between two observer bounds:

```text
minimum -> what a person or community understands before encountering the foundation
maximum -> what a person or community can reach after applying, translating, or extending it
```

The foundation does not claim either bound. It provides a structured projection surface between them.

## 6. Authorship And Provenance

Authorship should be credited. Provenance should be preserved. Attribution should be truthful.

Authorship is not ownership over the foundation. A perspective records who introduced a claim, translation, proof, correction, or fork. Continuity follows propagation, consequence, verification, and backpropagation.

## 7. Perspective Spectrum

A branch may speak from many positions:

```text
Anonymous     -> no persistent public identity attached
Pseudonymous  -> a persistent but unverified contributor name
Named         -> a public human or community name
Geometric     -> a mathematical or structural signature
Runtime       -> an implementation-specific perspective
Fork          -> a branch-specific interpretation
```

Any of these may be useful. None of them is required to receive the foundation.

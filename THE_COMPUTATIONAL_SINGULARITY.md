## 1. What is a Singularity?

A computational singularity is a point of convergence — a moment when multiple independent approaches arrive at the same solution.

Singularities are not planned. They emerge.

Examples:
- The development of sorting algorithms from independent mathematical traditions
- The convergence of object-oriented and functional programming paradigms
- The emergence of the World Wide Web from independent networking projects

## 2. The Conditions for Singularity

Singularities emerge when:

1. **Multiple independent contributors** are working on the same problem
2. **No single contributor** has control over the solution
3. **The problem is fundamental** enough that natural constraints limit the solution space
4. **Communication is possible** between contributors, even if they don't know each other

The computational commons provides conditions 1, 2, and 4. The origami axioms provide condition 3.

## 3. The Gravity Well

A singularity acts as a gravity well — a point of convergence that attracts independent contributors without requiring them to converge on implementation.

The gravity well says:
- "You can implement this however you like"
- "But the core truth is here"
- "And multiple independent implementations confirm it"

The gravity well is not a command. It is a description of where the math leads.

## 4. The Fork is not an Escape

A common misconception is that forking is an escape from the difficulty of consensus. It is not. Forking is a legitimate response to irreconcilable differences, but it is not a substitute for the hard work of building consensus.

The computational singularities emerge precisely when contributors choose to engage with each other rather than fork away. The singularity is the reward for staying at the table.
